Run a unit test for a project

Unit tests scrutinize the smallest testable parts of an application. When you create a SKY UX project, the template includes sample unit tests in the new project. You can follow the steps in this tutorial to run the unit test for the default single-page application.

For information about unit tests in SKY UX, see unit tests, and for information about how to test modals in SKY UX, see write unit tests for modals. For additional tips and techniques on testing Angular applications, see the Angular 2 testing guidelines.

  1. To view the unit test spec files in a local editor, open the project's about.component.spec.ts file in the \src\app\about folder and the home.component.spec.ts file in the \src\app folder.

    SKY UX unit tests should use the same name as the components or services that they test, but they should end with .spec.ts. The spec files should also be in the same folder as the components or services that they test.

  2. In the command prompt, if skyux serve is still running, enter Ctrl-C to stop serving the project.

  3. Enter skyux test to run the unit tests.

  4. The tests run, and the command prompt displays the results.

    When you run skyux test or skyux watch, code coverage results are stored in the coverage folder.
